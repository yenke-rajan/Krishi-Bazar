PRIORITY 2 — AUTOMATIC INVENTORY SYSTEM
═══════════════════════════════════
This is the core new feature. When an admin changes an order status,
the inventory_ledger must be updated automatically — no manual logging needed.

--- Backend: Update PATCH /api/orders/:id/status ---

Read server/src/routes/orders.ts first.

Current logic: validate status → update order status → return order.

NEW logic — add automatic inventory entries AFTER the status update,
inside the same database transaction:

IF new status === 'COLLECTED' (SUPPLY order, farmer's produce picked up):
  → The company has RECEIVED stock from the farmer.
  → Auto-insert into inventory_ledger:
    {
      id: uuidv4(),
      crop_id: order.crop_id,
      delta_quantity: order.weight_kg,
      tracking_type: 'RECEIVED',
      recorded_at: unixepoch(),
      notes: `Auto: Order ${order.order_id} collected from farmer`
    }

IF new status === 'DELIVERED' (DEMAND order, wholesaler received delivery):
  → The company has DELIVERED stock to the wholesaler.
  → Auto-insert into inventory_ledger:
    {
      id: uuidv4(),
      crop_id: order.crop_id,
      delta_quantity: order.weight_kg,
      tracking_type: 'DELIVERED',
      recorded_at: unixepoch(),
      notes: `Auto: Order ${order.order_id} delivered to wholesaler`
    }

For all other status changes: no inventory entry, just update the status.

IMPORTANT: Wrap the status update + inventory insert in a single SQLite 
transaction so they are atomic. If the inventory insert fails, 
the status update should also roll back.

Use this transaction pattern with better-sqlite3:
  const transaction = db.transaction(() => {
    // 1. UPDATE orders SET status = ? WHERE id = ?
    // 2. INSERT INTO inventory_ledger ... (only if COLLECTED or DELIVERED)
  });
  transaction();

Return in the API response an extra field `inventoryUpdated: true/false`
so the frontend knows when to refresh the inventory display.
